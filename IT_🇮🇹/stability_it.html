Quando un sistema √® caratterizzato da una funzione di trasferimento $ G(s) $, dove $ = G(s) $, ottenendo l‚Äôoutput desiderato $ Y $ in risposta ad un input $ R(s) $ non implica automaticamente che il sistema reale rispecchier√† questa risposta. Ci√≤ sottolinea l‚Äôimportanza fondamentale di condurre un‚Äôanalisi approfondita della stabilit√† prima di simulazioni o esperimenti pratici. √à fondamentale che tutte le variabili di stato si stabilizzino su uno stato stazionario per affermare la stabilit√† complessiva del sistema.

Comprendere il ruolo dei disturbi (\(w\)) e le condizioni iniziali (\(x_0\))

Il cambiamento del disturbo $ w $ modifica effettivamente le condizioni iniziali del sistema, alterando cos√¨ l‚Äôaccumulo di energia. Variando le condizioni iniziali nello spazio degli stati e osservando la stabilit√† di $ x_1, x_2, , x_n $, possiamo valutare la stabilit√† del sistema.

  1. Influenza dei disturbi (\(w\)):

    • Questi disturbi influiscono sull‚Äôaccumulo di energia all‚Äôinterno del sistema. Questo effetto √® simile all‚Äôalterazione delle condizioni iniziali del sistema (\(x_0\)).

  2. Condizioni iniziali e risposta del sistema:

    • Modificare le condizioni iniziali (\(x_0\)) e osservare la risposta del sistema permette di capire come i disturbi potrebbero influenzare la stabilit√† del sistema.
    • Variando le condizioni iniziali (\(x_0\)) in un ampio intervallo nello spazio degli stati, possiamo simulare come il sistema potrebbe comportarsi in diversi scenari.
    • Questo processo prevede l‚Äôesame della stabilit√† delle variabili di stato (\(x_1, x_2, ..., x_n\)) in queste diverse condizioni.
    • Se le variabili di stato rimangono stabili (cio√® non divergono) in questo intervallo di condizioni iniziali, possiamo essere fiduciosi nella stabilit√† complessiva del sistema.

In sintesi, il concetto di stabilit√† nei sistemi di controllo non riguarda solo l‚Äôosservazione dell‚Äôoutput del sistema ma implica un‚Äôanalisi completa di come le variabili di stato reagiscono ai cambiamenti delle condizioni iniziali e ai disturbi esterni. Esaminando attentamente la stabilit√† di queste variabili, possiamo prevedere e garantire prestazioni affidabili del sistema in condizioni reali.

Metodologia di analisi della stabilit√†

  • Stabilit√† BIBO (Bounded-Input Bounded-Output): questo concetto garantisce che se l‚Äôinput $ r $ √® limitato, anche l‚Äôoutput $ y $ sar√† limitato.

  • Stabilit√† zero input: questo aspetto si concentra sul comportamento del sistema in assenza di un input esterno, basandosi solo sulle condizioni e sui disturbi iniziali.

√à fondamentale operare all‚Äôinterno dell‚Äôintervallo lineare del sistema affinch√© questi concetti di stabilit√† siano validi. Negli intervalli non lineari, l‚Äôanalisi della stabilit√† diventa pi√π complessa e va oltre lo scopo della nostra discussione attuale.

import numpy as np
import matplotlib.pyplot as plt
import control as ctl

def simulate_mass_spring_damper(m, c, k, initial_displacement, initial_velocity):
    """
    Simulates the mass-spring-damper system and plots its response.

    Parameters:
    m (float): Mass
    c (float): Damping coefficient
    k (float): Spring constant
    initial_displacement (float): Initial displacement
    initial_velocity (float): Initial velocity
    """

    # System matrices
    A = [[0, 1], [-k/m, -c/m]]
    B = [[0], [1/m]]
    C = [1, 0]
    D = [0]

    # Create the state-space model
    sys = ctl.StateSpace(A, B, C, D)

    # Time array
    t = np.linspace(0, 10, 1000)

    # Initial state [displacement, velocity]
    X0 = [initial_displacement, initial_velocity]

    # Simulate the system
    t, y = ctl.initial_response(sys, T=t, X0=X0)

    # Plot the response
    plt.figure()
    plt.plot(t, y)
    plt.title('Mass-Spring-Damper System Response')
    plt.xlabel('Time (seconds)')
    plt.ylabel('Displacement (meters)')
    plt.grid(True)
    plt.show()

import numpy as np
import matplotlib.pyplot as plt
from matplotlib.animation import FuncAnimation
import control as ctl
from IPython.display import HTML

def draw_pendulum(ax, l, angle, is_upright):
    """
    Draws the pendulum on the given axes.

    Parameters:
    ax (matplotlib.axes.Axes): The axes to draw on.
    l (float): Length of the pendulum
    angle (float): Angle of the pendulum (in radians)
    is_upright (bool): Indicates if the pendulum is in the upright position
    """
    # Convert angle to Cartesian coordinates
    x = l * np.sin(angle)
    y = l * np.cos(angle) if is_upright else -l * np.cos(angle)

    # Clear previous drawing and draw the pendulum
    ax.clear()
    ax.plot([0, x], [0, y], color='blue', lw=2)
    ax.plot(x, y, 'o', color='red', markersize=10)

    # Set appropriate limits for both scenarios
    if is_upright:
        ax.set_ylim(-1.5*l, 1.5*l)
    else:
        ax.set_ylim(-1.5*l, 0)

    ax.set_xlim(-1.5*l, 1.5*l)
    ax.set_aspect('equal', adjustable='box')
    ax.grid(True)
    ax.set_title('Pendulum Animation')


def simulate_pendulum(g, l, initial_angle, is_upright):
    """
    Simulates a pendulum system and creates an animation of its response.

    Parameters:
    g (float): Acceleration due to gravity
    l (float): Length of the pendulum
    initial_angle (float): Initial angle (in radians)
    is_upright (bool): True for simulating the upright position, False for the downward position
    """
    # Pendulum parameters
    A = [[0, 1], [g/l if is_upright else -g/l, 0]]
    B = [[0], [1/l]]
    C = [1, 0]
    D = [0]

    # Create the state-space model
    sys = ctl.StateSpace(A, B, C, D)

    # Time array for simulation
    t = np.linspace(0, 1, 10)

    # Initial state [angle, angular velocity]
    X0 = [initial_angle, 0]

    # Simulate the system
    t, y = ctl.initial_response(sys, T=t, X0=X0)

    # Create figure and axes for the animation
    fig, ax = plt.subplots(figsize=(6, 6))

    # Function to update the animation
    def update(frame):
        draw_pendulum(ax, l, y[frame], is_upright)

    # Create animation
    ani = FuncAnimation(fig, update, frames=len(t), interval=10, repeat=False)  # Adjusted interval
#     plt.show()
    return ani

# Example usage
ani = simulate_pendulum(g=9.81, l=1.0, initial_angle=0.1, is_upright=True)  # Unstable (Upright)

Stabilit√† BIBO

La stabilit√† BIBO √® un concetto critico nello studio dei sistemi di controllo, in particolare per i sistemi lineari tempo-invarianti. Nella stabilit√† BIBO, consideriamo un sistema inizialmente rilassato, il che significa che al tempo $ t = 0 $, tutte le variabili di stato $ x $ sono a zero e il sistema non ha energia iniziale immagazzinata. Questo scenario fornisce un punto di riferimento per analizzare come il sistema risponde agli input esterni.

Tieni presente che \(t=0\) √® un orario di riferimento, per i sistemi Linear Time Invariant (LTI) √® l‚Äôorario iniziale. L‚Äôanalisi di stabilit√† per sistemi variabili nel tempo e/o non lineari √® pi√π complessa e va oltre lo scopo di questo corso.

Questa ipotesi semplifica l‚Äôanalisi della stabilit√† poich√© possiamo concentrarci sul modello della funzione di trasferimento per analizzare la stabilit√†. Per un sistema rilassato, l‚Äôenergia iniziale √® zero e questo significa che tutte le variabili di stato non sono eccitate e possiamo tranquillamente trascurarle. Poich√© tutte le condizioni iniziali sono zero, possiamo studiare la stabilit√† solo in termini della sua funzione di trasferimento.

Il principio chiave qui √® che se l‚Äôinput esterno $ r $ √® limitato, anche l‚Äôoutput $ y $ dovrebbe rimanere limitato.

Analisi quantitativa della stabilit√† BIBO

Il metodo quantitativo per stabilire la stabilit√† BIBO implica garantire che l‚Äôoutput $ y $ rimanga limitato per qualsiasi input limitato $ r $. Questo principio si applica indipendentemente dalla natura dell‚Äôinput, che si tratti di un comando desiderato o di un disturbo proveniente dall‚Äôambiente. La limitatezza dell‚Äôoutput √® un requisito fondamentale affinch√© un sistema di controllo possa essere considerato stabile secondo i criteri BIBO.

  • Nella stabilit√† BIBO, ci concentriamo sulla risposta del sistema agli input esterni (comando e disturbo) assumendo zero energia iniziale.
  • La stabilit√† pu√≤ essere analizzata utilizzando il modello della funzione di trasferimento, poich√© un sistema rilassato pu√≤ essere completamente descritto da questo modello.
  • L‚Äôobiettivo √® garantire che per ogni input limitato $ r $, l‚Äôoutput $ y $ rimanga limitato.

Esempio di stabilit√† BIBO

  • Consideriamo un sistema con ingresso a gradini. Se il passo in ingresso √® limitato, anche la risposta in uscita del sistema dovrebbe mostrare un comportamento limitato, non crescere indefinitamente.

Stabilit√† a ingresso zero

Il secondo aspetto del nostro studio sulla stabilit√† √® la stabilit√† con input zero, dove si presuppone che l‚Äôinput esterno sia zero.

Il modello matematico per la stabilit√† a ingresso zero √® rappresentato da:

\[ \dot{x} = Ax \]

dove $ x(t = 0) = x_0 $. L‚Äôattenzione qui √® se lo stato del sistema $ x(t) $ rimane limitato per tutti $ t $ quando varia $ x_0 $. Se √® cos√¨, il sistema preserva la stabilit√† a input zero.

Riepilogo

  • La stabilit√† con input zero considera la risposta del sistema quando si presuppone che gli input esterni siano pari a zero.
  • L‚Äôattenzione si concentra su come le variazioni nello stato iniziale $ x_0 $ influenzano lo stato del sistema $ x(t) $ nel tempo.
  • L‚Äôobiettivo √® garantire che gli stati del sistema rimangano vincolati per tutto il tempo $ t $ con input esterni pari a zero.

Chiamiamo un sistema invariante nel tempo in cui l‚Äôinput √® zero, un sistema autonomo. Questo concetto √® cruciale per studiare la stabilit√† a input zero.

Domanda pop-up: Perch√© la stabilit√† a input zero √® importante e in cosa differisce dalla stabilit√† BIBO?

Risposta: la stabilit√† a input zero si concentra sul comportamento del sistema in assenza di input esterni, enfatizzando la risposta del sistema in base al suo stato iniziale. Si differenzia dalla stabilit√† BIBO, che riguarda la risposta del sistema agli input limitati esterni, garantendo che l‚Äôoutput rimanga limitato.

  • Per un sistema lineare, se sono soddisfatte sia le condizioni BIBO che quelle di stabilit√† con input zero, si pu√≤ dedurre che per qualsiasi input limitato $ r $ e $ w $, gli stati e l‚Äôoutput del sistema rimarranno limitati.

Stabilit√† BIBO e risposta all‚Äôimpulso

  • La risposta all‚Äôimpulso $ g(t) $ di un sistema fornisce una caratterizzazione completa di un sistema rilassato. L‚Äôoutput del sistema √® dato da:

\[ y(t) = \int_{0}^{\infty} g(\tau)r(t-\tau) d\tau \]

dove \(r(t)\) √® l‚Äôinput.

Poich√© siamo interessati alla stabilit√† BIBO, ci√≤ significa che siamo interessati alle ampiezze. In questo caso vale la seguente relazione:

\[ |y(t)| = \Big| \int_{0}^{\infty} g(\tau)r(t-\tau) d\tau \Big| \le \int_{0}^{\infty} |g(\tau)| |r(t-\tau)| d\tau \]

Se l‚Äôinput √® limitato, ovvero \(|r(t-\tau)| \le M\), con \(M\) finito,

\[ |y(t)| = \Big| \int_{0}^{\infty} g(\tau)r(t-\tau) d\tau \Big| \le \int_{0}^{\infty} |g(\tau)| |r(t-\tau)| d\tau \le M \int_{0}^{\infty} |g(\tau)| d\tau \le N < \infty \]

Ci√≤ significa che per la stabilit√† BIBO, l‚Äôintegrale del modulo della risposta all‚Äôimpulso dovrebbe essere finito.

L‚Äôintegrale del modulo della risposta all‚Äôimpulso del sistema caratterizza la stabilit√† BIBO. Nello specifico, un sistema √® BIBO stabile se e solo se:

\[ \int_{0}^{\infty} |g(t)| dt < \infty \]

Questa condizione implica che l‚Äôarea sotto la curva del valore assoluto della risposta all‚Äôimpulso deve essere finita.

Poli, zeri e analisi delle funzioni di trasferimento

\[ \frac{Y(s)}{R(s)} = G(s) = \frac{b_0 s^m + b_1 s^{m-1} + \ldots + b_m}{a_0 s^n + a_1 s^{n-1} + \ldots + a_n} \]

  • L‚Äôequazione caratteristica, derivata dal denominatore della funzione di trasferimento, √®:

\[ \Delta(s) = a_0 s^n + a_1 s^{n-1} + \ldots + a_n = 0 \]
